table

type: widget/table

Overview:

A table widget is used to display data in a tabular format. It consists of a header row with column labels, followed by any number of rows containing data.The header may have a different color or image scheme than the rows, and the rows may or may not be configured to alternate between two colors or background images.

Table widgets come in three different flavors: static, dynamicpage, and dynamicrow. Static table widgets are built on the server and write their data directly into the container in which they reside, which is usually a scrollpane widget.= Dynamicpage table widgets load their data once they initialize on the client, by activating a query through an ObjectSource nonvisual widget.Dynamicpage table widgets do not support modification, but can be reloaded through an ObjectSource at will.Dynamicrow table widgets, on the other hand, display each row as an individual layer, and thus are modifiable on the client. Dynamicrow table widgets also load their contents through an ObjectSource widget query.As of the time of writing of this document, only static mode was supported.

Table widgets allow the selection (keyboard, mouse, and data focus) of individual rows.

Usage:

Table widgets are normally placed inside of a scrollpane so that any rows which don't fit int the container can still be viewed. Table columns are created via "widget/table-column" child widgets within the table widget.

Properties:

x integer x-coordinate of the upper left corner of the table. Default is 0.
y integer y-coordinate of the upper left corner of the table. Default is 0.
width integer width, in pixels, of the table.The height is determined dynamically.
bgcolor string A color, RGB or named, to be used between table cells and rows.If neither transparent.
background string A background image for the table. This "shows through" between table cells.
hdr_background string A background image for the header row cells.
hdr_bgcolor string A color, RGB or named, for the header row cells.
row_background1 string A background image for the table row cells.
row_bgcolor1 string A color, RGB or named, for the table row cells.
row_background2 string A background image for the table row cells. If this is specified, rows will alternate in "row_background2".
row_bgcolor2 string A color, RGB or named, for the table row cells. If this is specified, rows will alternate "row_bgcolor2".
row_backgroundhilight integer the background image of the highlighted row
row_bgcolorhilight integer the background color of the hilighted row
titlecolor (string A color, RGB or named, of the text in the header row. If unset, defaults to textcolor.
textcolor (string A color, RGB or named, of the text in the normal data rows.
textcolorhilight integer the text color of the hilighted row
inner_border integer width of the inner spacing between cells in a table. Default0.
inner_padding integer margins within each cell, in pixels. Default is 0 pixels.
outer_border integer width of the outer spacing around the outside of the table, in pixels. Default0.
rowheight integer The outside height of a row
cellhspacing integer The distance each cell is from the outside border of the row
cellvspacing integer the distance each cell is from the 'border' of the cell (each cell is 2*cellvspacing from it's neighbors)
windowsize integer maximum number of data rows to display at one time (windowsize of 5 = 1 title row + 5 data rows = 6 total rows)
sql string The SQL query to be used for obtaining the data for the table.This query's result set child widgets.
dragcols 0/1 boolean 0/1 -- indicates whether to enable dragable columns
colsep integer size of draggable column border/2 (probably should be renamed.
gridinemptyrows 0/1 boolean 0/1 -- if there are empty rows at the bottom (ie, windowsize=5, 2 rows in recordset), should the grid be there or not
followcurrent 0/1 boolean 0/1 -- should the table ensure that when the current osrc record changes, that it is visible in the dynamic table window?
mode integer 0=static, anything else=dynamic

Child Properties:

(of any child widgets)

title string The title of the column to be displayed in the header row.
width integer width of the column.

Actions:

none currently available

Events:

none currently available

Sample Code:

		
$Version=2$
// Here is a simple table.
tblFileList "widget/table"
	{
	sql = "select :name,annotation=condition(:annotation=='','-none-',:annotation)
				from /samples";
	mode="static";
	width=408;
	inner_border=2;
	inner_padding=1;
	bgcolor="#c0c0c0";
	row_bgcolor1="#e0e0e0";
	hdr_bgcolor="white";
	textcolor="black";

	name "widget/table-column" { title="Object Name";width=20; }
	annotation "widget/table-column" { title="Annotation"; width=25; }
	}

NEWS

01-Mar-2005: Version 0.7.4 released. Main features include prebuilt RPM's (tested on Fedora Core 2 and RedHat 7.3), a totally revamped report writer with PDF/PostScript support, several more widgets including the ObjectCanvas widget for data-driven diagrams and maps, moderate Mozilla/IE6 support, reliability enhancements, a POP3 driver, composite widgets, presentation hints (data display rules), and some internal DHTML generator enhancements in preparation for component support and more.

27-Aug-2004: Version 0.7.4 is in the process of being tested for release. Look forward to a number of new features and bug fixes, including moderate IE and Mozilla support, lots of widget improvements, a revamped report writer, "Presentation Hints" visual presentation rules support, composite widget support, and more!

24-Apr-2003: Version 0.7.3 is scheduled for release on May 31st. Components currently under active development include a completely revamped print formatter (for reporting), an NFS server (so you can mount the objectsystem itself!), and the "Presentation Hints" business rules component.

Centrallix....

Welcome to the website for the Centrallix(tm) application server! Centrallix is a Web-based application/reporting platform and data management engine featuring data abstraction, structural embedding, rich DHTML pages resembling a normal GUI, an SQL engine for multi-source queries, and declarative object-based development.


Project Status....

Centrallix currently consists of over 100,000 lines of highly modular C code, and depends heavily on JavaScript for the operation of the generated web pages. The system has many working components -- good enough for a small demo that won the ICCM Technology for Missions (TFM contest), but is still lacking in some important areas. The project was conceptualized in 1997 and has been underway off and on since late 1998, with a particularly concentrated development effort since February 2002.

  • Tested platforms: RedHat® Linux® 7.3 on x86, Fedora Core 2 on x86, Cygwin(tm) on Microsoft® Windows XP®, MacOS X(tm) on PPC, RedHat Linux 6.2 on SPARC(tm).
  • Server-side network drivers: HTTP, BDQS (under development), NFS (under development).
  • Content and client connectivity support: XML, Sybase®, CSV, HTTP, MIME, mailbox file, shell command, report writer, query tree, local filesystem, "lpr" printers, Centrallix structure files, gzipped content, Linux /dev/dsp, POP3, DBL ISAM files, BerkeleyDB files.
  • User interface widgets: checkbox, clock, connector (nonvisual), date-time, dropdown list, edit box (single line), server method exec (nonvisual), form (nonvisual, for field grouping), form status, frameset, html area, image, image button, label, menu, object canvas, objectsource (nonvisual, for server-side data integration), page, pane, radiobutton, scrollbar, scrollpane, spinner, tab control, table, textarea (multiline), text button, terminal (vt emulator), timer (nonvisual), treeview, variable (nonvisual), window.

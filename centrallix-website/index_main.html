NEWS

27-June-2002: Version 0.7.2 has been released. Changed include enhanced controls and a lot of added functionality. Thanks to all the developers who worked so diligently on the system to get it to where it is (even if they still think it isn't mature enough).

19-June-2002: Version 0.7.2 is under way. Centrallix/Kardia has gained a great deal of support since it's introduction and demonstration at the 2002 ICCM, causing a lot of anticipation as development speeds up even more.

17-Oct-2001: Version 0.7.1 released. This version is an interim development release focusing primarily on minor enhancements. We're continuing to press on towards the version 0.8.0 semi-stable release!

07-Aug-2001: The Centrallix source code is now in CVS and is "ready", more or less, for development to begin. We'll be making announcements at various times over the next month in regard to this.

20-Jun-2001: We've finally begun the process of launching Centrallix into the OpenSource arena! The decision to open-source this project came with much discussion and strategic planning, and we're confident that creating free open-source software from this system is the right thing to do. We're very open to any help, suggestions, and even the redesign of components if necessary.

Centrallix....

Welcome to the website for the Centrallix(tm) application server! Centrallix is a web-based app server and data management engine featuring data abstraction, structural embedding, "pro"-DHTML generation, a SQL engine for multi-source queries, and object-based development. That's the official definition... unofficially, we're attempting to build the ultimate web-based application server in terms of flexibility, structure, extensibility, usability for real-world business applications (as opposed to just websites), and perhaps more....


Project Status....

Centrallix currently consists of over 100,000 lines of highly modular C code, and depends heavily on JavaScript for the operation of the generated web pages. The system has many working components -- good enough for a small demo that won the ICCM Technology for Missions (TFM contest), but is still lacking in some important areas. The project was conceptualized in 1997 and has been underway off and on since late 1998, with a particularly concentrated development effort since February 2002.

  • Tested platforms: RedHat® Linux® 6.2 on x86, RedHat Linux 7.3 on x86, Cygwin(tm) on Microsoft® Windows XP®, MacOS X(tm) on PPC, RedHat Linux 6.2 on SPARC(tm).
  • Server-side network drivers: HTTP, (BDQS under development).
  • Content and client connectivity support: XML, Sybase®, CSV, HTTP, MIME, mailbox file, shell command, report writer, query tree, local filesystem, "lpr" printers, Centrallix structure files, gzipped content, Linux /dev/dsp.
  • User interface widgets: checkbox, clock, connector (nonvisual), date-time, dropdown list, edit box (single line), server method exec (nonvisual), form (nonvisual, for field grouping), form status, frameset, html area, image button, label, menu, objectsource (nonvisual, for server-side data integration), page, pane, radiobutton, scrollpane, spinner, tab control, table, textarea (multiline), text button, timer, treeview, variable (nonvisual), window.

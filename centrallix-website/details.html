Detailed Features....

Here's a few detailed features of the Centrallix system for those who are interested. This page can also serve to help explain some of the terminology used in Centrallix development.

  • Data Abstraction Layer (OSML - ObjectSystem Management Layer)
  • Partial Transaction Support (OXT - Object Transaction Tree)
  • SQL Engine (MQ - MultiQuery) with aggregate and sorting support
  • DHTML Generator (HTGEN - HTML Generator)
  • Report Writer (RPT - Reporting objectsystem driver)
  • Printed Output Formatter (PRT - Printing subsystem), currently includes Epson-FX, PCL-5, HTML, and plain text.
  • Multithreaded Infrastructure (MTASK - MultiTasking subsystem)
  • Query Objects (QY) for stored queries
  • QueryTree Objects (QYT) for translation of flat data into hierarchical data
  • CSV File Interpreter (DAT - flat Data file) handling variable-length ASCII delimited records
  • Sybase(r) connectivity (SYBD - CT-Library Sybase Driver)
  • HTTP Network Driver (NHT - Network HTTP Driver)
  • UN*X Filesystem Access (UXD)
  • UN*X Printer Access (UXP)
  • Widget drivers (some incomplete) include: checkbox, connector, html control, image button, pull-down / popup menus, pages, panes, radio buttons, framesets, scrollpanes, tab controls, tables, text buttons, draggable popup/dialog windows, and DHTML variables.
  • Annotations on all objects
  • Hierarchical Object Types (based on MIME types) allows specifying a more general type when opening an object to modify object-open behavior and OSD (objectsystem driver) invocation.

Coming Features....

Here are some features that (some) are under construction currently or (most) are desired to be implemented...

  • Forms, editboxes, drop-down-lists, and other form Widget controls.
  • A drawing canvas widget control
  • More widgets: driectory listing widgets, error/question channel window widget
  • Replication support, including replication to the UI.
  • New network interfaces: CORBA, CGI, HTTPS, NFS (mount the ObjectSystem!)
  • A rewrite of the Printed output formatter for greater flexibility
  • Object property sheet generators
  • Pseudo-filenames: '?' determine object properties without opening one, '*' create new object with auto-naming (or, auto primary key / numbering)
  • Triggers and stored procedures within Centrallix
  • Application remote-control facilities
  • JavaScript scripting integration on the server side within Centrallix objects
  • Events triggered by the underlying object as well as by the user
  • JDBC and ODBC connectivity drivers for Centrallix. Possibly implemented by emulating PostgreSQL's network interface and then using its JDBC and ODBC drivers.
  • Completion of Transaction support: explicit begin tran, commit, rollback
  • New ObjectSystem Drivers: pop3, imap, mailbox file, MIME message, ldap, http passthrough, ftp access, indexed file, XML, and others
  • New ObjectSystem Drivers for DB connectivity: mysql, PostgreSQL.
  • A graphical design IDE
  • Property sheet and icon registry
  • Browser-sensitive DHTML generator
  • Themeable DHTML generator
  • More Centrallix SQL extensions: UNION, cross-object OR clauses, HAVING clause, subqueries, and lists/vectors.
  • Centrallix native network protocol (BQDS, the Batchable Query Data Service).

ObjectSystem Driver Support....

Here is a list of currently implemented and supported objectsystem drivers in Centrallix, as well as a description of what those drivers do and their status.

DriverDescriptionDevelopment Status
objdrv_datafile (DAT)CSV and other flat ASCII delimited datafile driver.Currently supports most variable-length record ascii datafiles, both for reading and for limited updating/inserting. Has a builtin memory-based indexing mechanism to speed up accesses by row number.
objdrv_qytree (QYT)Query Tree driver.This driver has the ability to rearrange the presentation of data from flat RDBMS-style tabular structure (or other structures) into a hierarchical layout. Very useful for presenting recursive tables (where a foreign key in the table points to the table's own primary key) so that they can be viewed with a treeview DHTML widget.
objdrv_report (RPT)Report Writer.This driver generates reports from a given report specification in structure file format. Currently supports tables, free-form data, aggregates, multiple query modes (nested, parallel, multinested). Output formats currently are HTML, plain text, PCL, and Epson-FX data.
objdrv_struct (STX)Structure File driver.This driver is a very plain structure file driver which parses the file and makes it available in an object heirarchy format. It is absolutely central at present because most node objects and applications are structure files.
objdrv_sybase (SYBD)Sybase ASE CT-Lib Driver.This is the first SQL RDBMS driver that was written for Centrallix. Currently supports char, varchar, int/smallint/tinyint/bit, datetime, money, and float native data types. Currently allows for insert/update/delete/select, although does not yet support table creation.
objdrv_unixuser (UXU)Password database driver.This read-only driver allows for the looking up of username, userid, and gecos information in /etc/passwd via the getpwent() library calls. Does not access /etc/shadow. Useful for retrieving the full user description when listing directories and the like.
objdrv_ux (UXD)Filesystem Driver.This driver allows read/write access to a subtree of the local filesystem through the ObjectSystem. Permits access to several file attributes such as size, last_modification, etc. This driver is normally used as the ObjectSystem root, so /usr/local/etc/rootnode will contain a path to the root of the ObjectSystem.
objdrv_uxprint (UXP)Printer access driver.The uxprint driver allows access to printers on the local system, normally via "lpr". Currently only permits printing; does not allow creation or management of printers.
